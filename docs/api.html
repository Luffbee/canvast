/paint

Paint related operations

/paint/pixels put

Operations on pixels

put

Set some pixels to a specified color.

Pixels are addressed by base and offsets, the ith pixel is at base * BLOCKSIZE + offsets[i] .

put /paint/pixels

Set some pixels to a specified color.

Pixels are addressed by base and offsets, the ith pixel is at base * BLOCKSIZE + offsets[i] .

  • Request
  • Response

Headers

  • Cookie: required(string)

    Authentication cookie

Body

Media type: application/json

Type: object

Properties
  • color: required(string - pattern: ^[0-9A-F]{8}$)

    RGBA 256 color

    Example:

    A3A3A3FF
  • base: required(string - pattern: ^-?\d+,-?\d+$)

    Block position.

    A block position is the coordinate of the left bottom pixel divided by BLOCKSIZE.

    Example:

    314,-159
  • offsets: required(array of object - minItems: 1 - maxItems: 256)

    Items: offset

    • x: required(integer)
    • y: required(integer)

Example:

{
  "color": "A3A3A3FF",
  "base": "13A,-9F",
  "offsets": [
    {"x": 0, "y": 0},
    {"x": 1, "y": -1},
    {"x": 2, "y": -1}
  ]
}

/paint/blocks get put

Operations on blocks

get

Retrieve blocks in a rectangle that updated after the specified timestamp.

put

Set blocks in a rectangle.

Request body must be a zipfile which contains width * height png files with RGBA 256 color.
Each png file should named in block_{offset x}_{offset y}.png format,
where 0 <= offset x < width and 0 <= offset y < height .

get /paint/blocks

Retrieve blocks in a rectangle that updated after the specified timestamp.

  • Request
  • Response

Query Parameters

  • base: required(string - pattern: ^-?\d+,-?\d+$)

    Block position of the left bottem block

    Example:

    314,-159
  • width: required(integer - minimum: 1 - maximum: 128)

    Width of the rectangle

  • height: required(integer - minimum: 1 - maximum: 128)

    Height of the rectangle

  • timestamp: required(integer - minimum: 0)

    Timestamp

HTTP status code 200

A zipfile named in blocks_{new timestamp}.zip format.
The zipfile contains blocks have been updated after timestamp and before new timestamp .
Each block should be named in block_{offset x}_{offset y}.png format.

Body

Media type: application/zip

Type: file
